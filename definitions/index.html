    Definition

    Graphes

    • Graphe : Ensemble de sommets et d'arcs
    • Chaine (chemin en orienté) : Suite de sommets par lesquels passer, suite d'aretes consécutives reliant un sommet à un autre (dans un graphe non orienté)
    • Chaine Eulerienne : Chaine si tous les sommets sont pairs, ou tous sont pairs sauf 2 (pour faire une chaine passant par toutes les arretes une seule fois).
    • Graphe orienté : Graphe dont les arcs ont une direction
    • Graphe pondéré : Graphe dont les arcs ont un poids
    • Sommet : Élément d'un graphe pouvant être relié à d'autres sommets
    • Arete (arc en orienté) : Lien non orienté entre deux sommets
    • Degré entrant : Nombre d'arc dont la finalité est ce sommet
    • Degré sortant : Nombre d'arc dont la source est ce sommet
    • Degré (d'un sommet) : Somme des degrés entrant et sortant d'un sommet
    • Arbre : Graphe connexe sans cycle
    • Connexe (composante) : Graphe dans lequel il existe un chemin entre chaque paire de sommets
    • Composante (d'un graphe) : Sous-graphe
    • Fortement connexe (graphe) : Si tout le graphe est connexe
    • Graphe complet : Graphe dans lequel il existe un arc entre chaque paire de sommets
    • Graphe simple : Graphe sans boucle ni arc multiple
    • Graphe biparti : Graphe dont l'ensemble des sommets peut être divisé en deux sous-ensembles disjoints tels que chaque arc relie un sommet d'un sous-ensemble à un sommet de l'autre
    • Cycle (circuit pour orienté) : Chemin dont le sommet de départ est le même que le sommet d'arrivée
    • Chemin simple : Chemin sans sommet répété
    • Chemin élémentaire : Chemin simple sans arc répété
    • Chemin eulérien : Chemin passant par chaque arc exactement une fois
    • Circuit eulérien : Chemin eulérien dont le sommet de départ est le même que le sommet d'arrivée
    • Graphe eulérien : Graphe contenant un circuit eulérien
    • Graphe hamiltonien : Graphe contenant un cycle passant par chaque sommet exactement une fois
    • Graphe planaire : Graphe pouvant être dessiné dans le plan sans que deux arcs ne se croisent

    Algorithmique

    • Donnée : Valeur utilisée par un programme
    • Programme : ensemble d'instructions et d'opérations destinées à être exécutées par un ordinateur
    • Algorithme : Suite finie d'instructions permettant de résoudre un problème
    • Algorithme glouton : Algorithme qui résout un problème en faisant des choix locaux optimaux
    • Algorithme de force brute : Algorithme qui résout un problème en testant toutes les solutions possibles
    • Dictionnaire : Structure de données associant une clé à une valeur
    • File : Structure de données FIFO
    • Pile : Structure de données LIFO
    • Variable : Emplacement de mémoire contenant une valeur
    • Tableau : Structure de données contenant une séquence de valeurs de taille fixe
    • Liste : Structure de données contenant une séquence de valeurs de taille variable
    • Pointeur : Variable contenant l'adresse d'une autre variable
    • Boucle : Structure de contrôle répétant un bloc d'instructions
    • Condition : Structure de contrôle exécutant un bloc d'instructions si une condition est vraie
    • Booléen : Type de donnée pouvant prendre les valeurs vrai ou faux
    • Entier : Type de donnée représentant un nombre entier
    • Flottant : Type de donnée représentant un nombre à virgule flottante
    • Chaine de caractères : Type de donnée représentant une séquence de caractères
    • Fonction : une portion de code informatique nommée, qui accomplit une tâche spécifique
    • Procédure : Fonction ne retournant pas de valeur
    • Récursivité : Appel d'une fonction par elle-même
    • Tri : Algorithme réordonnant une séquence de valeurs
    • Recherche : Algorithme trouvant une valeur dans une séquence
    • Complexité : Mesure de la quantité de ressources nécessaires à l'exécution d'un algorithme
    • Calculabilité : Propriété d'un problème pouvant être résolu par un algorithme
    • Décidabilité : Propriété d'un problème pouvant être résolu par un algorithme donnant une réponse correcte
    • NP-complétude : Propriété d'un problème pour lequel il n'existe pas d'algorithme efficace connu
    • Heuristique : Algorithme donnant une solution approchée à un problème
    • Type : Catégorie de valeurs partageant des propriétés communes
    • Type de base : Type de donnée primitif
    • Complément à 2 : Représentation binaire d'un nombre négatif (inversion des bits et ajout de 1, ex : -3 = 1101 -> 0010 + 1 = 0011, c'est aussi $2^(n+1)$ - la valeur signée : 16 - 13)

    Base de données

    • Base de données : Ensemble de données organisées
    • Système de gestion de base de données (SGBD) : Logiciel permettant de stocker et de manipuler des données
    • MCD : Modèle Conceptuel de Données, schéma conceptuel d'une base de données
    • MLD : Modèle Logique de Données, schéma logique d'une base de données
    • Algèbre relationnelle : Langage permettant de manipuler des relations
    • SQL : Structured Query Language, langage permettant de manipuler des bases de données
    • Relation : Table de base de données, représentation d'une entité et de ses attributs
    • Attribut : Colonne d'une relation, caractéristique d'une entité
    • Clé primaire : Attribut unique identifiant une entité
    • Clé étrangère : Attribut faisant référence à une clé primaire d'une autre relation
    • Index : Structure de données accélérant la recherche dans une base de données
    • Jointure : Opération combinant les lignes de deux relations
    • Vue : Requête stockée dans une base de données
    • Transaction : Ensemble d'opérations devant être exécutées de manière atomique
    • ACID : Propriétés d'une transaction (Atomicité, Cohérence, Isolation, Durabilité)
    • Normalisation : Processus permettant de structurer une base de données

    Réseaux

    • Réseau : Ensemble d'ordinateurs interconnectés
    • Protocole : Ensemble de règles permettant la communication entre ordinateurs
    • Modèle OSI : Modèle de référence pour les réseaux informatiques (Open Systems Interconnection) :

    Couche physique : Transmission de bits Couche liaison : Transmission de trames Couche réseau : Transmission de paquets Couche transport : Transmission de segments Couche session : Gestion des sessions Couche présentation : Gestion des données Couche application : Applications

    • Modèle TCP/IP : Modèle de référence pour les réseaux informatiques (Transmission Control Protocol/Internet Protocol)
    • Adresse IP : Identifiant unique d'un ordinateur sur un réseau
    • Masque de sous-réseau : Valeur permettant de diviser une adresse IP en deux parties
    • Passerelle : Ordinateur permettant de relier deux réseaux
    • DNS : Domain Name System, système permettant de traduire un nom de domaine en adresse IP
    • DHCP : Dynamic Host Configuration Protocol, protocole permettant d'attribuer dynamiquement des adresses IP
    • NAT : Network Address Translation, technique permettant de traduire des adresses IP
    • Web : Service permettant de consulter des pages sur Internet
    • HTTP : HyperText Transfer Protocol, protocole permettant de transférer des pages web
    • Internet : Réseau mondial d'ordinateurs interconnectés
    • Intranet : Réseau interne à une organisation
    • Routeur : Appareil permettant de relier deux réseaux
    • Routage : Processus permettant de déterminer le chemin optimal entre deux ordinateurs
    • Switch : Appareil permettant de relier plusieurs ordinateurs (commutateur)

    Matériel

    • Ordinateur : Machine exécutant des programmes
    • Processeur : Composant d'un ordinateur exécutant des instructions
    • Mémoire : Composant d'un ordinateur stockant des données
    • Disque dur : Composant d'un ordinateur stockant des données de manière permanente
    • Carte mère : Composant d'un ordinateur connectant les autres composants
    • Carte graphique : (GPU) Composant d'un ordinateur traitant les calculs graphiques
    • Ordonnanceur : Composant d'un ordinateur gérant l'exécution des programmes
    • Système d'exploitation : Logiciel gérant les ressources d'un ordinateur
    • Logiciel : Programme exécuté par un ordinateur
    • IHM : Interface Homme-Machine, tout ce qui permet à un utilisateur de communiquer avec une machine
    • API : Interface de programmation d'application, ensemble de fonctions permettant à un programme d'interagir avec un autre programme
    • IDE : Environnement de développement intégré, logiciel regroupant des outils pour développer des programmes
    • Langage de programmation : Langage permettant d'écrire des programmes
    • Compilation : Transformation d'un programme écrit dans un langage de programmation en un programme exécutable
    • Interprétation : Exécution d'un programme écrit dans un langage de programmation sans transformation préalable
    • Language machine : Langage compréhensible par un processeur
    • Langage de haut niveau : Langage compréhensible par un humain
    • ROM : Mémoire morte, mémoire non volatile = mémoire de stockage
    • RAM : Mémoire vive, mémoire volatile = mémoire de travail
    • Cache : Mémoire rapide, mémoire intermédiaire entre la RAM et le processeur
    • Bus : Ensemble de fils reliant les composants d'un ordinateur (ex: bus de données, bus d'adresse)
    • Registre : Emplacement de mémoire dans un processeur
    • Horloge : Signal périodique régulant le fonctionnement d'un ordinateur
    • BIOS : Logiciel de base d'un ordinateur (Basic Input/Output System)
    • SATA : Interface de connexion pour les disques durs (Serial Advanced Technology Attachment)
    • USB : Interface de connexion pour les périphériques (Universal Serial Bus)
    • RISC : Architecture de processeur avec un jeu d'instructions réduit (Reduced Instruction Set Computing)
    • CISC : Architecture de processeur avec un jeu d'instructions complexe (Complex Instruction Set Computing)

    • Objet : Instance d'une classe
    • Classe : Modèle définissant les attributs et les méthodes d'un objet
    • Attribut : Caractéristique d'un objet
    • Méthode : Fonction associée à une classe
    • Héritage : Relation entre une classe et ses sous-classes
    • Polymorphisme : Capacité d'un objet à prendre plusieurs formes (ex: surcharge, redéfinition)
    • Surcharge : Définition de plusieurs méthodes avec le même nom mais des paramètres différents

    • Portée : Visibilité d'une variable ou d'une méthode (exemple : public, private, protected)

    Algorithmes

    • Dijkstra : Algorithme de recherche de plus court chemin dans un graphe :

    Initialisation : On attribue une distance infinie à tous les sommets sauf le sommet de départ, pour lequel on attribue une distance de 0 Boucle : On choisit le sommet non visité le plus proche du sommet de départ, on met à jour les distances des sommets voisins, et on marque le sommet choisi comme visité Arrêt : Lorsque tous les sommets ont été visités ou que le sommet d'arrivée a été visité

    def dijkstra(graph: Graph, start: Node, end: Node) -> List[float]:
    unvisited = set(graph.nodes)
    distances = {node: float('inf') for node in graph.nodes}
    distances[start] = 0
    while unvisited:
        current = min(unvisited, key=lambda node: distances[node])
        unvisited.remove(current)
        for neighbor in graph.neighbors(current):
            distances[neighbor] = min(distances[neighbor], distances[current] + graph.distance(current, neighbor))
    return distances[end]
    • Bellman-Ford : Algorithme de recherche de plus court chemin dans un graphe avec arcs de poids négatifs :

    Initialisation : On attribue une distance infinie à tous les sommets sauf le sommet de départ, pour lequel on attribue une distance de 0 Boucle : On met à jour les distances des sommets en parcourant tous les arcs du graphe Arrêt : Lorsque plus aucune distance n'est mise à jour

    def bellman_ford(graph: Graph, start: Node, end: Node) -> List[float]:
    distances = {node: float('inf') for node in graph.nodes}
    distances[start] = 0
    for _ in range(len(graph.nodes) - 1):
        for node in graph.nodes:
            for neighbor in graph.neighbors(node):
                distances[neighbor] = min(distances[neighbor], distances[node] + graph.distance(node, neighbor))
    return distances[end]
    • Kruskal : Algorithme de recherche d'arbre couvrant de poids minimum dans un graphe pondéré :

    Initialisation : On crée un arbre vide Boucle : On ajoute l'arc de poids minimum qui ne crée pas de cycle dans l'arbre Arrêt : Lorsque l'arbre contient n-1 arcs

    • Prim : Algorithme de recherche d'arbre couvrant de poids minimum dans un graphe pondéré :

    Initialisation : On crée un arbre contenant un sommet de départ Boucle : On ajoute l'arc de poids minimum reliant un sommet de l'arbre à un sommet hors de l'arbre Arrêt : Lorsque l'arbre contient n-1 arcs

    def prim(graph: Graph, start: Node) -> Graph:
    tree = Graph()
    tree.add_node(start)
    while len(tree.nodes) < len(graph.nodes):
        min_edge = None
        min_cost = float('inf')
        for node in tree.nodes:
            for neighbor in graph.neighbors(node):
                if neighbor not in tree.nodes and graph.distance(node, neighbor) < min_cost:
                    min_edge = (node, neighbor)
                    min_cost = graph.distance(node, neighbor)
        tree.add_node(min_edge[1])
        tree.add_edge(min_edge[0], min_edge[1], min_cost)
    return tree
    • A* : Algorithme de recherche de plus court chemin dans un graphe pondéré avec heuristique :

    Initialisation : On attribue une distance infinie à tous les sommets sauf le sommet de départ, pour lequel on attribue une distance de 0 Boucle : On choisit le sommet non visité le plus proche du sommet de départ en tenant compte de l'heuristique, on met à jour les distances des sommets voisins, et on marque le sommet choisi comme visité Arrêt : Lorsque tous les sommets ont été visités ou que le sommet d'arrivée a été visité

    def a_star(graph: Graph, start: Node, end: Node) -> List[Node]:
    open_set = PriorityQueue()
    open_set.put((0, start))
    came_from = {}
    g_score = {node: float('inf') for node in graph.nodes}
    g_score[start] = 0
    f_score = {node: float('inf') for node in graph.nodes}
    f_score[start] = heuristic(start, end)
    while not open_set.empty():
        current = open_set.get()[1]
        if current == end:
            return reconstruct_path(came_from, end)
        for neighbor in graph.neighbors(current):
            tentative_g_score = g_score[current] + graph.distance(current, neighbor)
            if tentative_g_score < g_score[neighbor]:
                came_from[neighbor] = current
                g_score[neighbor] = tentative_g_score
                f_score[neighbor] = g_score[neighbor] + heuristic(neighbor, end)
                open_set.put((f_score[neighbor], neighbor))
    return []
    • Tri bulle : Algorithme de tri réordonnant une séquence de valeurs en comparant les éléments deux à deux et en les permutant si nécessaire :
    def tri_bulle(tab):
    n = len(tab)
    for i in range(n):
        for j in range(0, n-i-1):
            if tab[j] > tab[j+1]:
                tab[j], tab[j+1] = tab[j+1], tab[j]
    return tab
    • Tri insertion : Algorithme de tri réordonnant une séquence de valeurs en insérant chaque élément à sa place dans une partie triée de la séquence :
    def tri_insertion(tab):
    for i in range(1, len(tab)):
        cle = tab[i]
        j = i-1
        while j >= 0 and cle < tab[j]:
            tab[j+1] = tab[j]
            j -= 1
        tab[j+1] = cle
    return tab
    • Tri sélection : Algorithme de tri réordonnant une séquence de valeurs en sélectionnant à chaque étape l'élément minimum et en le plaçant à sa position définitive :
    def tri_selection(tab):
    for i in range(len(tab)):
        min_idx = i
        for j in range(i+1, len(tab)):
            if tab[j] < tab[min_idx]:
                min_idx = j
        tab[i], tab[min_idx] = tab[min_idx], tab[i]
    return tab
    • Recherche séquentielle : Algorithme de recherche trouvant un élément dans une séquence en le comparant à chaque élément de la séquence :
    def recherche_sequentielle(tab, x):
    for i in range(len(tab)):
        if tab[i] == x:
            return i
    return -1
    • Recherche dichotomique : Algorithme de recherche trouvant un élément dans une séquence triée en comparant l'élément recherché à l'élément médian de la séquence :
    def recherche_dichotomique(tab, x):
    debut = 0
    fin = len(tab) - 1
    while debut <= fin:
        milieu = (debut + fin) // 2
        if tab[milieu] == x:
            return milieu
        elif tab[milieu] < x:
            debut = milieu + 1
        else:
            fin = milieu - 1
    return -1

    Parcours de graphe en largeur

    def parcours_largeur(graph: Graph, start: Node) -> List[Node]:
    visited = []
    queue = [start]
    while queue:
        node = queue.pop(0)
        if node not in visited:
            visited.append(node)
            neighbors = graph.neighbors(node)
            for neighbor in neighbors:
                queue.append(neighbor)
    return visited

    Parcours de graphe en profondeur

    def parcours_profondeur(graph: Graph, start: Node) -> List[Node]:
    visited = []
    stack = [start]
    while stack:
        node = stack.pop()
        if node not in visited:
            visited.append(node)
            neighbors = graph.neighbors(node)
            for neighbor in neighbors:
                stack.append(neighbor)
    return visited

def parcours_profondeur_recursif(graph: Graph, start: Node, visited: List[Node] = None) -> List[Node]:
    if visited is None:
        visited = []
    visited.append(start)
    neighbors = graph.neighbors(start)
    for neighbor in neighbors:
        if neighbor not in visited:
            parcours_profondeur_recursif(graph, neighbor, visited)
    return visited
