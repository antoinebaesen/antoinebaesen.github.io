    Tri en temps linéaire

    Introduction

    On connaît déjà les tris par comparaison qui utilisent des comparaisons entre les éléments du tableau à trier. Mais ces tris sont limités à une certaine complexité. On va voir ici des tris qui ne sont pas limités par cette complexité. Ils sont appelés tris en temps linéaires.

    C'est quoi un tri en temps linéaire ?

    Un tri en temps linéaire est un tri dont la complexité est en $O(n)$. C'est-à-dire que la complexité du tri est proportionnelle à la taille du tableau à trier. Ces tris ne fonctionnent pas sur n'importe quel type de données. Ils sont basés sur des hypothèses sur les données à trier, il faut donc bien choisir le tri en fonction des données à trier pour savoir si un de ces tris est adapté.

    Un exemple de tri en temps linéaire : le tri par dénombrement

    Principe

    Le tri par dénombrement est un tri en temps linéaire. On compte le nombre d'occurence dans le tableau de chaque valeurs possibles. (Il faut donc qu'on connaisse la valeur maximale du tableau) On met ces comptages dans un tableau à l'indice correspondant à la valeur, c'est à dire que le nombre d'occurence de la valeur 3 sera à l'indice 3 du tableau des comptages. Cela nous permet ensuite de reconstruire le tableau trié à partir des comptages en mettant autant de fois chaque valeur qu'elle apparait dans le tableau.

    C'est un tri efficace pour trier des données qui se répètent beaucoup si on connais la plage de valeurs possibles. On peut l'utiliser pour trier des entiers compris entre 0 et 100 par exemple.

    Exemple

    On veut trier le tableau suivant :

    6 0 2 0 1 3 4 6 1 3 2 1

    On compte le nombre d'occurrences de chaque élément :

    Élément 0 1 2 3 4 5 6
    Nombre d'occurrences 2 3 2 2 1 0 2

    Cela correspond à ce tableau : [2, 3, 2, 2, 1, 0, 2]

    On peut ensuite reconstruire le tableau trié en utilisant les comptages pour mettre les éléments dans le bon ordre autant de fois qu'ils apparaissent, on met 2 fois 0, 3 fois 1, 2 fois 2, etc... : 2*[0] + 3*[1] + 2*[2] + 2*[3] + 1*[4] + 0*[5] + 2*[6] = [0, 0, 1, 1, 1, 2, 2, 3, 3, 4, 6, 6]

    0 0 1 1 1 2 2 3 3 4 6 6

    Pour aller plus loin

    Ils existent de nombreux autres tris en complexités linéaires. On peut citer par exemple le tri par base. Il est basé sur le principe de décomposer les entiers à trier en base $k$ et de trier la liste des entiers en fonction de la valeur de chaque chiffre en partant du chiffre des unités.

    Note : On montre la plupart de nos tests avec des tableaux d'entiers. Mais on peut aussi utiliser ces tris pour trier des tableaux de chaînes de caractères par exemple. Il suffit de définir une fonction de comparaison adaptée.

    Exemple :

    On veut trier le tableau suivant :

    BAC RUE ROC MUR SUD COQ DUC RAT SAC

    On trie les mots selon le dernier caractère :

    BAC ROC DUC SAC SUD RUE COQ MUR RAT

    On trie les mots selon l'avant-dernier caractère :

    BAC SAC RAT ROC COQ DUC SUD RUE MUR

    On trie les mots selon le premier caractère :

    BAC COQ DUC MUR RAT ROC RUE SAC SUD
