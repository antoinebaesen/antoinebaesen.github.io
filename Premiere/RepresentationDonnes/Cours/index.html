    Codage et représentation des données

    Introduction

    Nous avons vu que les ordinateurs ne peuvent manipuler que des 0 et des 1. Pour représenter des nombres, des lettres ou des images, il faut donc trouver un moyen de les coder en binaire. C'est ce que nous allons voir dans ce chapitre.

    Codage des caractères

    ASCII

    Le code ASCII (American Standard Code for Information Interchange) est un code de caractères informatiques standardisé. Il permet de représenter les lettres de l'alphabet, les chiffres, les signes de ponctuation et les caractères spéciaux. Chaque caractère est codé sur 7 bits, ce qui permet de représenter 128 caractères différents.

    Par exemple, le code ASCII de la lettre 'A' est 65, celui de 'B' est 66, etc.

    Exemple : Voyons comment coder le mot "NSI" en ASCII. N S I 78 83 73 01001110 01010011 01001001

    Et voilà comment on peut le stocker en mémoire.

    Unicode

    Pour représenter des caractères de toutes les langues du monde, il faut plus de 128 caractères. C'est pourquoi on utilise de nouveaux standards comme Unicode. Unicode utilise 16 bits pour coder chaque caractère, ce qui permet de représenter 2^16 = 65536 caractères différents. On a tellment de caractères disponibles que mêmes les émojis sont codés en Unicode.

    Par exemple :

    Caractère Code Unicode
    A U+0041
    é U+00E9
    ☺ U+263A
    :dragon: U+1F409

    Pour des raisons pratiques, les 128 premiers caractères Unicode sont les mêmes que les 128 premiers caractères ASCII. Comme ça, on peut lire un texte ASCII avec un éditeur Unicode.

    UTF-8

    Mais on ne peut pas stocker un caractère Unicode directement en mémoire, il faut le coder en binaire. C'est là qu'intervient le format UTF-8.

    UTF-8 (Unicode Transformation Format) est un format de codage de caractères Unicode. Il est compatible avec le code ASCII, c'est-à-dire que les caractères ASCII sont codés sur un octet (8 bits) et les caractères Unicode sont codés sur plusieurs octets.

    Principe de codage

    • Les caractères ASCII sont codés sur un octet, c'est-à-dire que le bit de poids fort (celui le plus à gauche) est toujours à 0.
    • Les caractères Unicode sont codés sur plusieurs octets. Le premier octet commence par autant de "1" qu'il y a d'octets utilisés pour écrire le nombre. A leur suite, on place un "0" pour indiquer la séparation entre ces "1" et l'encodage du caractère. Les octets suivants commencent par 10... pour indiquer qu'il s'agit d'un octet suivant. On rempli ensuite les bits restants avec les bits du caractère. Et les vides avec des 0.
    • Les caractères Unicode sont codés sur 1 à 4 octets.
    Exemple : Codons le mot "NSI" en UTF-8.
N  S  I => 01001110 01010011 01001001

    Exemple : Codons le caractère 'é' en UTF-8.
é correspond à U+00E9
00E9 = 0000.0000 1110.1001
On complète avec des 0 à gauche pour avoir 16 bits : 0000 0000.0000 1110.1001
On a besoin de 2 octets pour coder ce caractère.

Donc on écrira sous la forme 110xxxxx 10xxxxxx
On remplace les x par les bits de notre caractère : 11000011 10111001
Ce qui nous donne : 110__00011__ 10__111001__

    Pratique

    On veut inscrire le mot "Gaïa" sur un support numérique. Pour cela on va le coder en UTF-8.

    Un dernier problème

    Nous savons maintenant coder des nombres, des lettres et des caractères spéciaux en binaire. Mais nous avons un problème pour les nombres à virgule. Ils nous posent beaucoup plus de problèmes que les nombres entiers.

    Exercice 4

    En suivant une des deux méthodes que nous avons appris pour coder un nombre en binaire, tentez de convertir 3.14 en binaire.

    Quel problème rencontrez-vous ?

    ....................................................................................................

    Solution

    Pour pallier à ce problème, on va faire 2 choses : - Utiliser une valeur approchée du nombre à virgule que l'on veut coder. - Et utiliser la notation scientifique. On code le signe, l'exposant et la mantisse.

    Le signe : 0 pour un nombre positif, 1 pour un nombre négatif. L'exposant : On code l'exposant en binaire. Pour un nombre en simple précision, on utilise 8 bits pour coder l'exposant. Pour un nombre en double précision, on utilise 11 bits. La mantisse : On code la mantisse en binaire. Pour un nombre en simple précision, on utilise 23 bits pour coder la mantisse. Pour un nombre en double précision, on utilise 52 bits.

    Au total on va noter le nombre en binaire sur 32 ou 64 bits.

    Signe Exposant Mantisse
    1 bit 8 bits 23 bits

    Exemple : Codons le nombre 3.14 en simple précision.

    Le signe : 0 car le nombre est positif. L'exposant : On décale la virgule jusqu'au premier bit (le plus à gauche) valant 1. Déplacer la virgule à gauche revient à diviser le nombre par 2. Déplacer la virgule à droite revient à multiplier le nombre par 2. Ici, on décale la virgule de 1 bit vers la droite. Donc l'exposant est 1. La mantisse : On code la partie décimale du nombre en binaire. Ici, 0.14 = 0.00100110011001100110011... On a donc : 0.00100110011001100110011 = 1.00100110011001100110011 * 2^-1 Donc on a : | Signe | Exposant | Mantisse | |:-----:|:--------:|:--------:| | 1 bit| 8 bits | 23 bits | | 0 | 00000001 | 00100110011001100110011 |
