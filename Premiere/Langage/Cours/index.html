    Langages et programmation - Les constructions élémentaires

    On va faire quelques rappels de seconde sur les langages de programmation et les constructions élémentaires.

    Variables

    Une variable est un espace de mémoire qui permet de stocker une valeur. En Python, on peut déclarer une variable en lui affectant une valeur. Par exemple, pour déclarer une variable a qui contient la valeur 5, on écrit :

    unNombre = 5
duTexte = "Coucou"
interrupteur = True

    Types de variables

    Il existe plusieurs types de variables en Python. Les plus courants sont :

    • int : pour les entiers
    • float : pour les nombres à virgule
    • str : pour les chaînes de caractères. Une chaîne de caractères est déclarée avec des guillemets, par exemple "Bonjour"
    • bool : pour les booléens (vrai ou faux). En Python, on utilise True et False pour les booléens.
    • list : pour les listes. Une liste est déclarée avec des crochets, par exemple [1, 2, 3]
    • tuple : pour les tuples. Un tuple est déclaré avec des parenthèses, par exemple (1, 2, 3)

    Fonctions

    Une fonction est un bloc de code qui permet d'effectuer une tâche précise.

    Format d'une fonction

    def nom_de_la_fonction(parametre1 : type1, parametre2 : type2, ...) -> type_de_retour:
    # Instructions
    return valeur

    En Python, on définit une fonction avec le mot-clé def suivis du nom de notre fonction, des paramètre entre parenthèses et enfin, de ":". Toutes les lignes par la suite au sein de la fonction devront être indentées, c'est à dire décalé d'une tabulation. Le mot clé return permet de renvoyer une valeur.     Exemple

    def carre(x : int) -> int:
    return x * x

    Documentation d'une fonction

    Une fonction bien faite se doit d'être bien documentée. Cela va permettre de comprendre ce que fait la fonction, quels paramètres elle prend en entrée et ce qu'elle renvoie. En Python, la documentation complète d'une fonction doit intégrer 4 éléments obligatoirement (et un cinquième facultatif) :

    • Une description de la fonction
    • Les paramètres d'entrée avec leur type et leur signification
    • La valeur de retour avec son type et sa signification
    • Des exemples d'utilisation
    • Les préconditions s'il y en a, c'est à dire les conditions que les paramètres doivent respecter pour que la fonction se comporte correctement. (facultatif)

    Pour notre fonction carre, la documentation ressemblera à ceci :

    
def carre(x : int) -> int:
    """
    Renvoie le carré d'un nombre.

    Paramètres :
        x (int) : Le nombre dont on veut calculer le carré.

    Retour :
        int : Le carré de x.

    Exemple :
        >>> carre(5)
        25

        >>> carre(-3)
        9
    """
    return x * x

    Comme vous pouvez le voir, la documentation peut prendre une place importante dans le code et elle peut paraitre fastidieuse à écrire. Mais bien documenter votre code va vous permettre de gagner énormément de temps quand vous allez vouloir relire du code ou le modifier. Ca vous permet aussi de vous assurer que vous comprenez bien ce que vous avez écrit.
